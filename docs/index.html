Get started

Installation

Choose the installation method that works best for you.

Windows Bundle — Recommended for Windows users

  1. Download the Windows Bundle from the releases page
  2. Extract the ZIP directly to your drive root — e.g. C:\MikhMonCE-Windows\ or D:\MikhMonCE-Windows\
  3. Open the extracted folder and double-click MikhMonCE_Server.exe
  4. The server starts automatically and minimizes to the system tray
  5. Right-click the tray icon and select Open MikhMon
  6. Login with admin / admin — change password immediately after first login
✅ PHP 8.3 is included in the bundle — no additional software needed.
⚠ Note: If the launcher fails to start, install Visual C++ Redistributable 2019 x64 from Microsoft.

Standard — For existing web server users

  1. Download the Standard version from the releases page
  2. Extract the ZIP file
  3. Copy the mikhmon-ce folder to your web server root:
    # Laragon (Windows)
    C:\laragon\www\

    # XAMPP (Windows)
    C:\xampp\htdocs\

    # WAMP (Windows)
    C:\wamp64\www\
  4. Open your browser and go to http://localhost/mikhmon-ce
  5. Login with admin / admin

Linux — Apache + PHP

  1. Install Apache and PHP:
    sudo apt install apache2 php libapache2-mod-php php-curl php-mbstring php-sockets
  2. Download and copy MikhMon CE:
    sudo cp -r mikhmon-ce /var/www/html/
    sudo chown -R www-data:www-data /var/www/html/mikhmon-ce
  3. Restart Apache:
    sudo systemctl restart apache2
  4. Open your browser and go to http://localhost/mikhmon-ce
  5. Login with admin / admin

macOS — MAMP or Homebrew

  1. Install MAMP or PHP via Homebrew:
    brew install php
  2. Copy the mikhmon-ce folder to your web root:
    # MAMP
    /Applications/MAMP/htdocs/

    # Homebrew
    /opt/homebrew/var/www/
  3. Start your web server and open http://localhost/mikhmon-ce
  4. Login with admin / admin
Requirements: PHP 8.0 or higher  |  MikroTik API enabled: Winbox → IP → Services → api (port 8728)
Common setup

Guides

Essential guides for setting up MikhMon CE correctly with your MikroTik device.

🕐 Fix Date & Time After Router Reboot Essential

After a power outage or reboot, MikroTik resets the clock to Jan/01/1970 00:00:00. This causes MikhMon CE to malfunction — voucher expiry, session tracking and sales reports all depend on accurate time. This guide shows how to preserve the date and time across reboots.

Step 1 — Create a Script

Go to System → Script and create a new script:

  • → Name: datetime
  • → Leave On Event empty — it will be filled by the scheduler

Step 2 — Create a Scheduler (saves time every 5 minutes)

Go to System → Scheduler and create:

  • → Name: datetime
  • → Interval: 00:05:00
  • → On Event (paste this script):
    :local date [/sys clock get date];
    :local time [/sys clock get time];
    /sys scr set source="/sys clock set date=$date time=$time" [find where name=datetime];

Step 3 — Create a Startup Scheduler (restores time on boot)

Create another scheduler:

  • → Name: datetime-startup
  • → Start Time: startup
  • → Interval: 00:00:00
  • → On Event:
    /sys scr run datetime
How it works: The datetime(2) scheduler saves the current date and time to the datetime(1) script every 5 minutes. On reboot, datetime-startup runs the datetime(1) script which restores the last saved time. Once internet is available, SNTP Client updates to the accurate current time automatically.

Step 4 — Configure SNTP Client

Go to System → SNTP Client:

  • → Check Enabled
  • → Add Server DNS Names: asia.pool.ntp.org (or your preferred NTP server)

Also go to IP → Cloud:

  • → Uncheck Update Time — let SNTP Client handle time sync instead

And go to System → Clock:

  • → Uncheck Time Zone Autodetect
  • → Set your correct Time Zone manually (e.g. Asia/Manila for Philippines)
⚠ Recommended NTP Servers:
• pool.ntp.org — Global
• asia.pool.ntp.org — Asia region (recommended for Philippines)
• time.google.com — Google Public NTP
• time.nist.gov — NIST Internet Time Servers

🔌 Enable MikroTik API Service Essential

MikhMon CE communicates with your MikroTik device via the RouterOS API. This must be enabled before MikhMon CE can connect.

  • → Open Winbox and connect to your MikroTik
  • → Go to IP → Services
  • → Find api in the list and make sure it is enabled
  • → Default port is 8728 — leave as-is unless you have a specific reason to change it
  • → Optionally restrict access to specific IP addresses for security
✅ Once the API is enabled, enter your MikroTik's IP address, username and password in MikhMon CE's Session Settings and click Connect.

📶 Basic Hotspot Setup on MikroTik Recommended

Before using MikhMon CE, make sure your MikroTik hotspot is properly configured. Here is a quick checklist:

  • → Hotspot must be enabled on the interface connected to your clients — IP → Hotspot → Hotspot Setup
  • → Create at least one hotspot user profile — IP → Hotspot → User Profiles
  • → Make sure DNS is working on your MikroTik — IP → DNS
  • → Enable API service as described above
  • → Note your MikroTik's local IP address (usually 192.168.88.1)
⚠ Important: MikhMon CE does NOT use a radius server. It communicates directly with MikroTik via the RouterOS API on port 8728.

🔐 Change Default Login Credentials Essential

MikhMon CE ships with default credentials. Change them immediately after first login.

  • → Login with admin / admin
  • → Go to Admin Settings (gear icon in the sidebar)
  • → Change your Username and Password
  • → Click Save
⚠ Before updating MikhMon CE: Always back up your include/config.php file before replacing files with a new release. This file stores your credentials and will be overwritten if not backed up.
