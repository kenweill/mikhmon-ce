Get started

Installation

Choose the installation method that works best for you. Default login is admin / admin — change it immediately after first login.

Windows Bundle — Recommended for Windows users

  1. Download the Windows Bundle from the releases page
  2. Extract the ZIP to your drive root — e.g. C:\MikhMonCE-Windows\ or D:\MikhMonCE-Windows\
  3. Open the extracted folder and double-click MikhMonCE_Server.exe
  4. Windows may show a SmartScreen warning — click More info then Run anyway
  5. The server starts automatically and minimizes to the system tray
  6. Right-click the tray icon and select Open MikhMon
  7. Login with admin / admin
PHP 8.3 is included — no additional software needed.
If the launcher fails to start: Install Visual C++ Redistributable 2019 x64 from Microsoft, then try again.

Standard — For existing web server users

  1. Download the Standard version from the releases page
  2. Extract the ZIP file
  3. Copy the mikhmon-ce folder to your web server root:
    # Laragon (Windows)
    C:\laragon\www\

    # XAMPP (Windows)
    C:\xampp\htdocs\

    # WAMP (Windows)
    C:\wamp64\www\
  4. Open your browser and go to http://localhost/mikhmon-ce
  5. Login with admin / admin

Linux — Apache + PHP

  1. Install Apache and PHP:
    sudo apt install apache2 php libapache2-mod-php php-curl php-mbstring php-sockets
  2. Copy MikhMon CE to the web root:
    sudo cp -r mikhmon-ce /var/www/html/
    sudo chown -R www-data:www-data /var/www/html/mikhmon-ce
  3. Restart Apache:
    sudo systemctl restart apache2
  4. Open http://localhost/mikhmon-ce and login

macOS — MAMP or Homebrew

  1. Install MAMP or PHP via Homebrew:
    brew install php
  2. Copy mikhmon-ce to your web root:
    # MAMP
    /Applications/MAMP/htdocs/

    # Homebrew
    /opt/homebrew/var/www/
  3. Start your web server and open http://localhost/mikhmon-ce
  4. Login with admin / admin
Requirements: PHP 8.0 or higher  ·  MikroTik API enabled: Winbox → IP → Services → api (port 8728)
Common setup

Setup Guides

Essential guides for setting up MikhMon CE correctly with your MikroTik device.

Fix Date & Time After Router Reboot

Essential

After a power outage or reboot, MikroTik resets its clock to Jan/01/1970 00:00:00. This causes MikhMon CE to malfunction — voucher expiry, session tracking and sales reports all depend on accurate time. This guide shows how to preserve the date and time across reboots.

Step 1 — Create a Script

Go to System → Script and create a new script:

  • → Name: datetime
  • → Leave On Event blank — it will be filled by the scheduler
Step 2 — Create a Scheduler (saves time every 5 minutes)

Go to System → Scheduler and create:

  • → Name: datetime
  • → Interval: 00:05:00
  • → On Event — paste this script:
:local date [/sys clock get date];
:local time [/sys clock get time];
/sys scr set source="/sys clock set date=$date time=$time" [find where name=datetime];
Step 3 — Create a Startup Scheduler (restores time on boot)

Create another scheduler:

  • → Name: datetime-startup
  • → Start Time: startup
  • → Interval: 00:00:00
  • → On Event: /sys scr run datetime
How it works: The scheduler saves the current time to the datetime script every 5 minutes. On reboot, datetime-startup restores the last saved time. Once internet is available, SNTP Client updates to the accurate time automatically.
Step 4 — Configure SNTP Client

Go to System → SNTP Client:

  • → Check Enabled
  • → Server DNS Names: asia.pool.ntp.org (recommended for Asia/Philippines)

Go to IP → Cloud:

  • → Uncheck Update Time — let SNTP Client handle time sync

Go to System → Clock:

  • → Uncheck Time Zone Autodetect
  • → Set your correct timezone manually (e.g. Asia/Manila for Philippines)
Recommended NTP servers:
pool.ntp.org · asia.pool.ntp.org · time.google.com · time.nist.gov

Enable MikroTik API Service

Essential

MikhMon CE communicates with your MikroTik via the RouterOS API. This must be enabled before MikhMon CE can connect.

  • → Open Winbox and connect to your MikroTik
  • → Go to IP → Services
  • → Find api in the list and make sure it is enabled
  • → Default port is 8728 — leave as-is
  • → Optionally restrict access to specific IP addresses for security
Once the API is enabled, enter your MikroTik IP, username and password in MikhMon CE's Session Settings and click Connect.

Change Default Login Credentials

Essential

MikhMon CE ships with default credentials. Change them immediately after first login.

  • → Login with admin / admin
  • → Go to Admin Settings in the sidebar
  • → Change your Username and Password
  • → Click Save
Before updating MikhMon CE: Always back up include/config.php before replacing files with a new release. This file stores your credentials and will be overwritten if not backed up.

Basic Hotspot Setup on MikroTik

Recommended

Before using MikhMon CE, make sure your MikroTik hotspot is properly configured.

  • → Enable hotspot on the interface connected to your clients — IP → Hotspot → Hotspot Setup
  • → Create at least one user profile — IP → Hotspot → User Profiles
  • → Make sure DNS is configured — IP → DNS
  • → Enable API service as described above
  • → Note your MikroTik's local IP address (usually 192.168.88.1)
Note: MikhMon CE does NOT use a radius server. It communicates directly with MikroTik via the RouterOS API on port 8728.

Troubleshooting

Common issues and how to fix them.

⚠ Windows protected your PC — SmartScreen warning

Windows Defender SmartScreen shows a warning when running MikhMonCE_Server.exe for the first time. This is expected for unsigned executables and the app is safe to run.

Fix: Click More info (small link below the warning text) then click Run anyway. Windows will remember your choice and will not show this warning again.

⚠ php.exe - Application Error (0xc000007b)

The launcher fails to start with an application error.

Fix: Install Visual C++ Redistributable 2019 x64 from Microsoft. PHP 8.3 requires this runtime to be installed.

⚠ Cannot connect to MikroTik

MikhMon CE shows a connection error or ping timeout.

Fix: Make sure the API service is enabled on your MikroTik (IP → Services → api, port 8728). If running on shared hosting, outbound TCP on port 8728 may be blocked — use a local installation instead.

⚠ Date and time reset to Jan/01/1970 after reboot

MikroTik clock resets after every power outage or reboot causing voucher and session issues.

Fix: Follow the Fix Date & Time After Router Reboot guide above. This is a very common issue with MikroTik devices that do not have a hardware RTC battery.

⚠ Login failed after updating MikhMon CE

After updating to a new release, the username and password no longer work.

Fix: The update overwrote include/config.php which stores your credentials. Always back up this file before updating. To recover, login with the default admin / admin and set a new password in Admin Settings.

⚠ Port already in use — server won't start

MikhMonCE_Server.exe cannot start because the port is occupied by another application.

Fix: Open MikhMonCE_Server.exe, change the port in Settings (e.g. 8080 or 8181) and click Change Port. The server will restart on the new port.
